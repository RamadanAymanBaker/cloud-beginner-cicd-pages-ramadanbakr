What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It is a development practice where code changes are automatically tested and deployed to production. Continuous Integration helps developers merge code frequently while Continuous Deployment automatically publishes the latest version of the project. This process reduces manual work and helps teams deliver updates faster and more reliably.

My Pipeline

In this project, when I push my code to the main branch on GitHub, GitHub Actions automatically runs the workflow. The workflow checks if the required files exist and then deploys the website to GitHub Pages. After the workflow finishes successfully, the updated website becomes available online.
